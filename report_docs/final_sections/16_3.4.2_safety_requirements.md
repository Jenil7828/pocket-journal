# 3.4.2 Safety Requirements

- Data integrity: All writes to Firestore must be confirmed and logged; partial failures must be detected and retried.
- Failure handling: Model failures should not result in loss of user data; fallbacks (truncated summary, empty mood) should be returned with logging for monitoring.
- Data retention: Interactions and embeddings retention configurable (default 90–180 days) and implementable via scheduled archival jobs.
- Safe defaults: LLM insights generation is opt-in; default configuration does not expose raw LLM responses to end users.

These safety measures are implemented in the service layer and operational scripts to ensure user data remains available and recoverable in the face of partial failures.
