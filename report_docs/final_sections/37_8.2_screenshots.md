# 8.2 Screenshots (Representative UI and Outputs)

This file contains textual descriptions of representative screenshots and example UI outputs; image files are not included in this repository.

1. Journal List View
   - Display: Paginated list of entries showing `title`, `created_at`, `summary` and dominant mood icon (derived from `entry_analysis.mood`).
   - Behavior: Date-range filter and mood filter controls.

2. Entry Detail View
   - Display: Full `entry_text`, abstractive `summary`, detailed mood probability vector and embedded semantic tags.
   - Behavior: Buttons for re-analysis, edit and delete.

3. Recommendations View (Movies)
   - Display: 10-item paginated list with title, poster, genre, and concise rationale for recommendation (constructed from intent vector and mood).

4. Insights View
   - Display: Structured insight document with sections `Goals`, `Progress`, `Negative Behaviors`, `Remedies`, `Appreciation`, `Conflicts`.
   - Behavior: Option to export insight or view contributing entries via `insight_entry_mapping`.

5. Health Endpoint Response (JSON)
   - Example: `{"status": "ok", "models": {"mood_detection": "loaded", "summarization": "loaded"}, "db": "connected"}`
