# 6.1 Overview of Project Modules

This section enumerates the primary code modules and their responsibilities, providing direct mappings to repository paths for implementation traceability.

1. `app.py` — Flask entrypoint: initializes services, loads config and registers routes.
2. `routes/` — Contains domain-based route modules: `journal_domain.py`, `media_domain.py`, `insights_domain.py`, `user.py`, `auth.py`, `health.py`, `export_route.py`.
3. `services/` — Business logic organized by domain: `journal_entries`, `media_recommender`, `insights_service`, `embeddings`, `personalization`, `export_service`, `stats_service`.
4. `ml/inference/` — Model predictor implementations: `mood_detection/roberta`, `summarization/bart`, `insight_generation/gemini` and `qwen2`.
5. `persistence/` — `db_manager.py` (DBManager) and `database_schema.py` (schema validation and examples).
6. `utils/` — Logging utilities, Firestore serialization and configuration loaders.
7. `scripts/` — Operational utilities: model download, cache refresh and entrypoint shell script for container deployments.

Each module is designed for testability and is referenced in the academic extraction documents for reproducibility and evaluation.
