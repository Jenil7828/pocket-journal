# 3.4.3 Security Requirements

- Authentication: All protected endpoints validate Firebase ID tokens; anonymous endpoints limited to health and home pages.
- Authorization: uid-based checks in service and DB layers; Firestore security rules mirror application-level checks.
- Encryption: TLS for all network traffic; Firestore-managed encryption at rest.
- Secrets management: Service account JSON files and provider API keys stored in `Backend/secrets/` and environment variables; not committed to source control.
- Audit logging: Significant operations (entry creation, insights generation, deletion) logged with uid and trace ID for auditing.
- Rate limiting: Interaction endpoints enforce per-user hourly limits to reduce abuse.

These requirements are realized by combining Firebase authentication, Firestore rules and server-side checks in route handlers.
