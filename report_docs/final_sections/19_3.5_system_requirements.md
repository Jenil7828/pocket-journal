# 3.5 System Requirements

This section consolidates the system, database, software and hardware requirements needed to deploy and evaluate the Pocket Journal system.

## 3.5.1 Database Requirements
- Firestore database with collections: `journal_entries`, `entry_analysis`, `journal_embeddings`, `user_vectors`, `insights`, `insight_entry_mapping`, `users`, `user_interactions`, `media_cache_*`.
- Composite indexes for queries involving `uid` and `created_at` as documented in `10_database.md`.

## 3.5.2 Software Requirements
- Python 3.10+ runtime
- Libraries: firebase-admin, torch, transformers, sentence-transformers, flask, requests
- Optional: Ollama or local LLM runtime for Qwen2 inference

## 3.5.3 Hardware Requirements
- Development: CPU with 8 vCPUs and 16 GB RAM
- Production (recommended): GPU instance with ≥16 GB VRAM for inference (BART, local LLM). Storage for models and cache.

These requirements support both development and production deployments and map directly to configuration parameters in `config.yml`.
