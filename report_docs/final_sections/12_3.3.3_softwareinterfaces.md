# 3.3.3 Software Interfaces

The system interfaces with several software components and libraries. This section documents the critical software interfaces, library versions (where applicable), and integration points.

1. Firebase Admin SDK (Python)
   - Purpose: Authentication token verification, Firestore access.
   - Interface: `firebase_admin` library; Firestore client operations via `db.db.collection(...)`.

2. PyTorch and HuggingFace Transformers
   - Purpose: Model loading and inference for RoBERTa, BART and optional local LLMs.
   - Interface: `transformers` library predictor wrappers defined under `ml/inference/*/predictor.py`.

3. Sentence-Transformers
   - Purpose: Embedding generation using `all-mpnet-base-v2`.
   - Interface: `services.embeddings.embedding_service.embed_text()` returns normalized 384-D vector.

4. External provider APIs
   - TMDb, Spotify, Google Books provider wrappers under `services/media_recommender/providers/*`.
   - Require API keys configured in environment variables or `config.yml`.

5. Local LLM runtime (optional)
   - Ollama or HuggingFace for Qwen2 local inference; HTTP or Python interfaces depending on chosen backend.

6. HTTP Framework
   - Flask routes defined under `routes/` with `@login_required` decorators and request/response logging utilities.

Operational integration notes:
- All model calls are encapsulated in predictor classes enabling mocking in tests.
- External API wrappers implement retry/backoff and timeouts driven by `config.yml` provider settings.

This software interfaces mapping supports testability and modular replacement of third-party components.
