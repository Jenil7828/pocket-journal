# 3.6 Analysis Models and SDLC to be Applied

## Analysis Models
- Data-driven evaluation: Use cross-validation for ML models (RoBERTa, BART) and standard metrics (precision, recall, F1, ROUGE-L).
- User-centric evaluation: Collect interaction-based metrics (CTR) and human judgments for summary quality and insight usefulness.
- Logging and offline analysis: Persist raw inputs, summaries, moods and embeddings for reproducible analysis.

## Software Development Lifecycle (SDLC)
- Version control: Git with feature branches and code reviews.
- Continuous integration: Run unit tests and linting on every PR; execute integration tests on merges.
- Experimentation lifecycle: Record `config.yml`, model versions and seeds for each experiment; maintain an experiments log.
- Release process: Tagged releases include model artifact references and deployment scripts.

This approach ensures rigorous development and controlled experimental evaluation aligned with academic reproducibility standards.
