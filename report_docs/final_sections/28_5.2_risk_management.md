# 5.2 Risk Management

## Identified Risks
1. Model unavailability or failure during inference
   - Mitigation: Implement CPU fallback and truncated-summary fallbacks; monitor model health and alert.

2. External provider API rate limits or downtime
   - Mitigation: Maintain media cache collections refreshed periodically; implement retry/backoff and serve cached results.

3. Data privacy and accidental exposure
   - Mitigation: Enforce Firestore security rules, store sensitive service account keys securely, implement opt-in for LLM insights.

4. Cold start for personalization
   - Mitigation: Cold-start handler uses default normalized user vectors and popular cache items; consider onboarding flows to collect initial preferences.

5. Cost overruns for cloud LLM usage
   - Mitigation: Provide configuration to disable cloud LLMs and use local Qwen2; track and report LLM API usage.

## Monitoring and Alerts
- Implement alerts for model load failures, high latency and Firestore error rates; log anomalous inference outputs for review.

## Contingency Plans
- If cloud LLMs are unavailable long-term, provide degraded insights using rule-based summarization and statistical aggregation.

Risk management is operationalised via the `scripts/` utilities and monitoring hooks in the codebase.
