# 7.1 Types of Testing

The project employs multiple testing levels to ensure correctness, robustness and reproducibility.

1. Unit testing
   - Target: services, utilities, predictor wrappers.
   - Approach: Mock external dependencies (Firestore client, model predictors).

2. Integration testing
   - Target: Service-to-database and service-to-model interactions.
   - Approach: Use Firestore emulator and fixture model outputs for validation.

3. System testing / end-to-end
   - Target: Full HTTP API flows including entry creation, recommendations and insights.
   - Approach: Deploy application in test environment and execute representative scenarios.

4. Performance testing
   - Target: Latency and throughput for recommendation and inference pipelines.
   - Tools: k6, Locust or similar load-testing frameworks.

5. Security and privacy testing
   - Target: Authorization enforcement, data isolation and secrets handling.
   - Approach: Automated tests asserting uid-based access and penetration test scenarios.

Each testing type is implemented with CI integration to ensure regression protection and reproducibility.
