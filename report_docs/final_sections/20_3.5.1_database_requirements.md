# 3.5.1 Database Requirements

- Firestore document database with support for composite indexes and efficient per-user filtering.
- Collections and schema as specified in `12_database.md`.
- Backup/restore procedures and retention policies configurable via operational scripts.
- Performance: low-latency reads for recent entries and efficient writes for high-volume entry creation.

Indexing recommendations:
- Compound index on (`uid`, `created_at`) for journal listing and time-range queries.
- Index on `entry_id` in `entry_analysis` and `journal_embeddings` for rapid lookup.

Operational requirements:
- Service account with appropriate Firestore permissions and secured JSON credential storage.
- Monitoring for read/write rates and cost control mechanisms for high-volume usage.
