# 3.2 Functional Requirements

This section lists the system's functional requirements derived from the implemented routes and services. Each requirement is traceable to a route or service module.

FR1 Create Journal Entry — `POST /api/v1/journal` (services/journal_entries/entry_create.py)
FR2 List Journal Entries — `GET /api/v1/journal` (services/journal_entries/entry_read.py)
FR3 Search Journal Entries — `GET /api/v1/journal/search` (route-level filtering)
FR4 Get Single Entry — `GET /api/v1/journal/{entry_id}`
FR5 Update Entry Content — `PUT /api/v1/journal/{entry_id}`
FR6 Delete Entry — `DELETE /api/v1/journal/{entry_id}`

FR7 Mood Detection — RoBERTa predictor (`ml/inference/mood_detection/roberta/predictor.py`)
FR8 Summary Generation — BART summarizer (`ml/inference/summarization/bart/predictor.py`)
FR9 Generate Insights — `POST /api/v1/insights/generate` (services/insights_service)
FR10 List Insights — `GET /api/v1/insights`
FR11 Get Single Insight — `GET /api/v1/insights/{insight_id}`
FR12 Delete Insight — `DELETE /api/v1/insights/{insight_id}`

FR13–FR16 Media Recommendations — domain-specific recommend endpoints (`routes/media_domain.py`) for movies, songs, books, podcasts using the unified recommendation pipeline (`services/media_recommender/recommendation_pipeline.py`).
FR17 Media Search — `GET /api/v1/{media_type}/search` (SearchService)
FR18 Track User Interaction — `POST /api/v1/media/interaction` (services/personalization/interaction_service.py)

FR19–FR20 User settings endpoints — `GET/PUT /api/v1/user/settings` (users collection)
FR21 Data Export — `GET /api/v1/export/data` (export_service)
FR22 Health Check — `GET /api/v1/health` (system/health_service)

Each functional requirement must be accompanied by input validation, authentication and authorization checks, and appropriate success and error responses as implemented in the route handlers and services.
