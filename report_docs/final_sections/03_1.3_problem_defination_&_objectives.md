# 1.3 Problem Definition and Objectives

## Problem Definition

The implemented system addresses five concrete gaps observed when converting free-form personal text into machine-actionable artifacts. These gaps span the full lifecycle: ingestion (journaling), per-entry analysis, personalization (recommendation) and longitudinal insight synthesis:

0) Unreliable ingestion and lack of canonical storage
   - Journal submissions may be malformed, missing provenance or inconsistently indexed. Without a robust ingestion and storage layer, downstream analysis cannot be reliably linked to source text and temporal queries become brittle.

1) Unstructured narratives lack machine-readable emotional representation
   - Long-form entries contain affective information distributed across sentences and clauses. Without per-entry probabilistic emotional quantification, downstream processes (aggregation, filtering, and personalized recommendations) cannot reliably operate on affective signals.

2) Lengthy textual entries impede rapid review and indexing
   - Users and analysts require condensed, semantically-faithful representations to scan past material efficiently. Raw text or keyword lists insufficiently preserve discourse-level information.

3) Absence of short-term contextual signals in personalization
   - Personalization systems that rely only on historical consumption fail to adapt to transient states captured in recent narrative entries (for example, a sudden change in mood or topic).

4) Lack of structured, multi-entry interpretations suitable for action
   - Synthesizing goals, progress, recurring negative behaviours and remedies from multiple entries requires both aggregation of per-entry analytics and generation of validated structured outputs.

Each of the above problems is addressed by a corresponding implemented capability in the system; the Objectives below formalize measurable targets that map to those capabilities.

## Objectives (Functionally Mapped and Measurable)

Objective 0 — Robust journaling ingestion and storage
  - Description: Accept, validate and persist user-provided journal submissions with strong provenance and temporal indexing.
  - Input: raw journal text and optional title; authenticated user context; operational metadata for ingestion.
  - Output: canonical persisted record with stable identifier, timestamps and integrity metadata; indices to support time-range and user-scoped queries.
  - Validation metric: data integrity checks (write success rate), ingestion throughput (records per second) and completeness (ratio of entries with analytic artifacts produced within a specified window).
  - Failure modes: partial writes, missing timestamps or index creation failures; ingestion errors are recorded and surfaced for operational remediation.
  - Justification:
    - Why: reliable ingestion and storage are foundational; without them analysis, recommendations and insights cannot be reproducibly derived.
    - What it solves: ensures canonical evidence and auditability for all downstream analytic operations.
    - If removed: downstream components would lose authoritative linkage to source text, compromising reproducibility and traceability.


Objective 1 — Probabilistic multi-label emotion detection
  - Description: For every stored entry, produce a probability distribution across seven discrete emotion categories (anger, disgust, fear, happy, neutral, sad, surprise).
  - Input: raw journal text normalized and tokenized to a maximum of approximately 128 tokens for this analysis stage.
  - Output: a persisted probability vector mapping each emotion category to a floating-point probability in [0,1], together with the threshold value used for any binary decision; these are recorded as analysis metadata.
  - Validation metric: report per-label precision, recall and macro F1 on a held-out annotated journal corpus; target macro F1 is specified as a design objective (documented as an evaluation target rather than an asserted measurement).
  - Failure modes: empty or excessively short entries result in low-confidence probabilities; in cases where the classifier is unavailable the system returns an explicit indicator and null probabilities.
  - Justification:
    - Why: probabilistic, multi-label outputs capture intensity and co-occurrence of affective signals, which single-label approaches cannot represent.
    - What it solves: provides graded affective inputs for filtering, personalization and longitudinal trend analysis.
    - If removed: aggregation and recommendation components would lack nuanced affective information and their precision would degrade.

Objective 2 — Abstractive summarization preserving semantic fidelity
  - Description: Transform long journal entries into concise summaries (20–128 tokens) that preserve salient events and factual coherence.
  - Input: journal text truncated to generation input limits when necessary (practical truncation near 1024 tokens); short texts below the minimum length are returned unchanged.
  - Output: a concise summary string and associated generation metadata describing output length and whether a fallback truncation was applied.
  - Validation metric: ROUGE-L and human-rated coherence on a representative sample; production responses record generation metadata for offline validation.
  - Failure modes: generation may produce overly generic or truncated summaries; system records fallback summaries when full generation is not possible.
  - Justification:
    - Why: abstractive summarization condenses narrative into semantically-rich short forms suitable for human review and automated embedding.
    - What it solves: reduces reading time and standardizes input size for subsequent embedding generation and similarity computations.
    - If removed: downstream modules would operate on variable-length, noisy text increasing computational cost and reducing consistency of similarity measures.

Objective 3 — Context-aware recommendation via blended intent and ranking
  - Description: Generate ranked, paginated media recommendations that reflect both stable user preferences and immediate context derived from recent entries.
  - Input: the user identity and optional selectors such as genre, mood label, free-text query and language; processing uses the most recent entry embedding and the user's persistent preference vector when available.
  - Processing and scoring: the system constructs an intent vector by blending persistent preference and recent entry embedding with heavy weighting toward historical preference. Candidate items are scored by cosine similarity to the intent and re-ranked by a composite scoring function that combines relevance, historical interaction frequency, item popularity and recency; a maximal marginal relevance step is applied to promote diversity.
  - Output: a paginated list of media item metadata (embeddings and internal scores omitted), accompanied by pagination metadata and an echo of applied filters.
  - Validation metric: offline evaluation of relevance (precision@k, recall@k, MRR) and online proxies such as click-through rate (CTR); diversity is measured as unique-item ratio.
  - Failure modes: cold-start users without embeddings or preference vectors receive default recommendations drawn from cached popular items; missing embeddings are handled as zero vectors.
  - Justification:
    - Why: blending intent ensures recommendations are simultaneously relevant to long-term taste and immediate context.
    - What it solves: reconciles stability and responsiveness in personalization while preserving diversity.
    - If removed: the system would either ignore transient context or overfit to short-term noise, reducing user engagement.

Objective 4 — Structured insight synthesis across time windows
  - Description: For a specified date range, aggregate per-entry summaries and emotion distributions and produce a validated structured document containing identified goals, progress observations, recurring negative behaviours, suggested remedies, expressions of appreciation and detected conflicts.
  - Input: a date window and the user identity; the pipeline consumes the stored per-entry summaries and emotion probability vectors within that window.
  - Output: a validated structured document containing the required interpretive fields together with metadata such as the number of contributing entries and the generation timestamp.
  - Validation metric: human evaluation for actionability and schema validation to ensure required fields are present; configured latency targets are recorded to monitor responsiveness.
  - Failure modes: malformed generation output triggers parsing fallback and returns an explicit error; the system persists raw responses for audit when parsing is successful.
  - Justification:
    - Why: structured insights synthesize longitudinal signals into actionable outputs for users and research.
    - What it solves: automates multi-entry interpretation and provides reproducible summaries for empirical studies.
    - If removed: longitudinal interpretation would require manual curation, increasing time cost and reducing reproducibility.

Objective 5 — Persistent analytic artifacts with provenance
  - Description: Persist summaries, emotion probability vectors and embeddings together with metadata specifying model configuration used for generation to enable reproducible offline evaluation.
  - Output: persisted analytic artifacts together with attached generation metadata (model reference, generation parameters and timestamps) to enable reproducibility.
  - Justification:
    - Why: artifact provenance permits controlled re-runs and comparison across model variants.
    - What it solves: prevents drift in experimental conditions and supports auditability.
    - If removed: experimental reproducibility would be significantly impaired and audit trails would be incomplete.

Each objective above is directly executable given the system's implemented input/output contracts and analytic artifact persistence. Objectives are expressed as measurable evaluation targets; where empirical measurements are not present in the repository the objective is stated as a design target to be validated by experiments described in the testing and results sections.
