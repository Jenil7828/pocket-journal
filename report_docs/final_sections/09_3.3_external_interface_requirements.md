# 3.3 External Interface Requirements

This section documents interfaces between the system and external actors or systems: user-facing APIs, hardware and software interfaces, and communication protocols.

## 3.3.1 User Interface (UI)
- The system exposes a RESTful JSON API. The UI (not part of this repository) is expected to authenticate users via Firebase and consume endpoints documented in `13_api_interfaces.md`.
- Input constraints: textual entry length ≤ 5000 characters; titles ≤ 200 characters.

## 3.3.2 Hardware Interfaces
- GPU: Optional NVIDIA CUDA device for model acceleration. Supported via PyTorch device selection.
- CPU-only operation supported for inference at lower throughput.

## 3.3.3 Software Interfaces
- Firebase Admin SDK for authentication and Firestore access.
- HuggingFace Transformers and PyTorch for model inference.
- Optional Ollama or HuggingFace inference API for local LLM usage.

## 3.3.4 Communication Interfaces
- HTTP/HTTPS for all client-server and provider interactions.
- OAuth2-style token exchange for provider APIs (Spotify) when live provider access is enabled.
- Internal services communicate via function calls; long-running background jobs use shell scripts and scheduler hooks documented in `scripts/`.

These interfaces are realized in code within `routes/`, `persistence/` and `services/` modules and require explicit configuration in `config.yml` and environment variables.
