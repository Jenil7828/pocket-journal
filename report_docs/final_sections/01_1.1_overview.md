# 1.1 Overview — Multi-Component Intelligent Journaling System

This document summarises the implemented system as four tightly integrated but distinct layers: (1) Journaling Layer, (2) Analysis Layer, (3) Recommendation Layer and (4) Insight Layer. Each layer's responsibilities, inputs, outputs, constraints and justifications are described to emphasise balanced importance across system capabilities.

1. Journaling Layer

1.1 Responsibility
- Accept, validate and persist user-submitted journal entries as canonical records with provenance and temporal indexing.

1.2 Input
- User-authored journal text (required) and optional title; authenticated user context and optional client metadata.

1.3 Transformations
- Validation (presence, type and length enforcement), normalization (whitespace, control-character handling) and canonical persistence with assigned entry identifier and timestamps.
- Index creation to support efficient time-range and user-scoped retrieval.
- Emission of ingestion events to trigger asynchronous analytic tasks where appropriate.

1.4 Output
- Canonical entry record with provenance metadata and indices that constitute the authoritative source for all downstream processing.

1.5 Justification and constraints
- A robust journaling layer ensures traceability and reproducibility. Operational constraints (e.g., configured maximum text length) balance storage and processing cost against fidelity.

2. Analysis Layer

2.1 Responsibility
- Produce per-entry analytic artifacts that capture different representational modalities: human-readable summarisation, graded affective signals and compact semantic embeddings.

2.2 Subcomponents and responsibilities
- 2.2.1 Summarization: produce a concise abstractive summary that preserves salient events and coherence for quick review and as input to embedding.
- 2.2.2 Emotion quantification: compute calibrated probabilities across implemented emotion dimensions to capture graded affective states as one component of analysis.
- 2.2.3 Embedding generation: convert the summary (or canonical text when necessary) into a fixed-dimensional numeric vector for efficient similarity-based operations.

2.3 Inputs and outputs
- Input: canonical entry text and ingestion metadata.
- Outputs: persisted summary with generation metadata, a per-dimension probability vector with thresholds and confidence indicators, and a normalized embedding vector with provenance.

2.4 Justification and constraints
- The three artifacts are complementary: summaries support human interpretation and synthesis, probability vectors capture affective nuance and embeddings enable scalable similarity and retrieval. Input truncation and generation limits bound latency and cost; the system records generation metadata to support offline analysis of limitations and fallbacks.

3. Recommendation Layer

3.1 Responsibility
- Produce ranked, diverse media recommendations that reflect both long-term user preference and immediate contextual signals derived from recent journaling.

3.2 Key steps
- 3.2.1 Intent formation: blend persistent preference vectors with the most recent entry-derived embedding to form a normalized intent representation, with conservative weighting to preserve historical preference.
- 3.2.2 Candidate retrieval and hard filtering: obtain a candidate set from maintained catalogs, applying explicit filters (genre, language, mood) and fallback relaxations when necessary.
- 3.2.3 Scoring and ranking: compute cosine similarity to intent as base relevance, combine with interaction frequency, popularity and recency into a hybrid score, and apply temporal decay to older interactions.
- 3.2.4 Diversity promotion: apply Maximal Marginal Relevance (MMR) to balance relevance and novelty across the returned list.

3.3 Outputs and client contract
- A paginated list of recommended items (internal fields redacted) with pagination metadata and explicit flags indicating fallback behaviour (e.g., cold-start or constrained candidate set).

3.4 Justification and edge cases
- Blending short- and long-term signals ensures recommendations are responsive yet stable. Edge cases (cold-start, insufficient candidates) are handled by default lists and filter relaxation with explicit client indicators.

4. Insight Layer

4.1 Responsibility
- Synthesize structured, validated interpretations across multiple entries within a user-selected window to produce actionable documents for reflection and research.

4.2 Key steps
- 4.2.1 Aggregation: retrieve per-entry summaries and probability vectors for the selected window and compute aggregate statistics (entry counts, mean per-emotion probabilities, high-probability counts).
- 4.2.2 Context assembly: select a bounded set of high-salience summaries by recency and computed importance to construct a context for synthesis.
- 4.2.3 Structured synthesis and validation: generate a schema-conformant insight document (goals, progress, recurring behaviours, remedies, appreciations, conflicts); validate the result and apply parsing fallbacks if necessary.
- 4.2.4 Persistence and provenance: persist validated insights with links to contributing entries and generation metadata to enable traceability.

4.3 Outputs and justification
- A validated insight document with required interpretive fields and provenance metadata. Structured insights convert per-entry signals into coherent, auditable narratives and are essential for actionable reflection and reproducible research.

Cross-layer considerations
- Each analytic artifact includes provenance metadata (generation parameters, model reference and timestamps) to enable reproducibility.
- The system is designed so failures in optional analytic components do not prevent canonical storage; absent artifacts are recorded and responses indicate missing components explicitly.
- The four layers are intended to be comparably important: journaling provides the authoritative record, analysis produces versatile artifacts, recommendation operationalises intent for personalised experiences and insight synthesises longitudinal understanding.
