# 3.3.1 UI Requirements

The system provides a RESTful API intended for consumption by web or mobile clients. UI requirements include:

- Authentication: UI must present Firebase ID token in `Authorization: Bearer <token>` header for protected endpoints.
- Entry creation form: Text area for `entry_text` (max 5000 chars), optional title, submit action that POSTs to `/api/v1/journal`.
- Journal listing: Paginated view calling `GET /api/v1/journal` with filtering controls for date range and mood.
- Entry detail: Display `summary`, `mood` probabilities, and additional metadata fetched from `GET /api/v1/journal/{entry_id}`.
- Recommendations view: Per-media-type recommendation pages calling endpoints such as `/api/v1/movies/recommend`.
- Insights view: Generate insights via `POST /api/v1/insights/generate` and display structured fields (`goals`, `progress`, `remedies`).

The UI must validate inputs locally (length checks, date formats) before sending requests and gracefully handle API error responses according to documented status codes.
