# 3.3.4 Communication Interfaces

## Protocols and Endpoints
- Primary protocol: HTTPS for all API calls (client ↔ server and server ↔ providers).
- REST endpoints documented in `13_api_interfaces.md` provide JSON request/response semantics.

## Authentication
- Firebase ID token in `Authorization: Bearer <token>` header.
- Server verifies tokens at route entry via `@login_required` decorator and middleware which attaches `request.user` to the request context.

## External Provider Communication
- External provider calls (TMDb, Spotify, Google Books) use HTTP(S) with timeout and retry logic configured in `config.yml`.
- Provider fallback: If cache insufficient, request live provider; otherwise serve cached items.

## Internal Communication
- Services within the same process communicate via direct method calls; long-running tasks may be executed via scripts (`scripts/`) or scheduled jobs with documented procedures.

## Rate Limiting and Quotas
- Interaction signals are rate-limited: `interactions.rate_limit_per_hour` (default 10 per media type per hour).
- Provider-level quotas managed via caching and request backoff.

These communication interfaces ensure secure client interaction and resilient integration with external services.
