# 6.3 Algorithms

This section summarizes the core algorithms used in the system with implementation references.

1. RoBERTa-based multi-label classification (Mood Detection)
   - Model: RoBERTa-base fine-tuned for seven emotion labels.
   - Implementation: `ml/inference/mood_detection/roberta/predictor.py`.
   - Inference: Tokenize → forward pass → sigmoid → probabilities → thresholding (default 0.25).

2. BART-based abstractive summarization
   - Model: `facebook/bart-large-cnn` fine-tuned for journal summaries.
   - Implementation: `ml/inference/summarization/bart/predictor.py`.
   - Generation parameters: `num_beams=4`, `min_length=20`, `max_length=128`, `no_repeat_ngram_size=3`.

3. Sentence embedding (All-MPNet-Base-V2)
   - Model: `sentence-transformers/all-mpnet-base-v2`.
   - Implementation: `services/embeddings/embedding_service.py`.
   - Output: 384-D L2-normalized vector used for similarity and intent construction.

4. Recommendation Ranking (Phase 5)
   - Components: intent blending (95% taste, 5% journal), cosine similarity, MMR (λ=0.7), temporal decay (rate=0.15/day), hybrid scoring (similarity=0.5, freq=0.2, pop=0.2, recency=0.1).
   - Implementation: `services/media_recommender/enhanced_ranking_engine.py` and `advanced_ranking.py`.

5. Insight Synthesis (LLM)
   - Backends: Gemini (cloud) or Qwen2 (local).
   - Process: aggregate summaries and mood distributions → structured prompt → LLM inference → parse and validate JSON output.
   - Implementation: `ml/inference/insight_generation/*/insight_analyzer.py`.

These algorithmic components are described in detail in `11_algorithms.md` for reproducibility and parameter tracing.
