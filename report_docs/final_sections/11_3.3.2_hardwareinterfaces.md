# 3.3.2 Hardware Interfaces

## Supported Hardware
- NVIDIA GPUs with CUDA support (recommended for production inference): enables float16 inference for RoBERTa and BART models.
- CPU-only environments supported for development and low-throughput inference.
- Disk storage for model artifacts and cache (models stored in MODEL_STORE_PATH if configured).

## Hardware Requirements (Representative)
- GPU instance: 16–32 GB VRAM recommended for BART and local LLM inferencing (larger models may require more VRAM).
- CPU instance: 8+ vCPUs and 16+ GB RAM for moderate throughput.
- Storage: 50+ GB for models and cached media assets depending on configured model store.

## Device Selection
- The codebase uses an auto-device selection pattern: if CUDA is available, models are moved to GPU; otherwise CPU is used. Configuration flags in `config.yml` control precision (use_half_precision) and device preference.

## Operational Notes
- For local Qwen2 inference, ensure Ollama or the HuggingFace runtime is correctly provisioned and accessible at the configured endpoint (e.g., `http://localhost:11434` for Ollama).
- Provide GPU instances for heavy batch inference (insight generation, summarization) to meet latency targets.
