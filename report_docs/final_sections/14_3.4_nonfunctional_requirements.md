# 3.4 Non-Functional Requirements

This section summarizes non-functional requirements derived from system design and `config.yml` settings.

1. Performance
   - Recommendation p95 latency ≤ 500 ms for 10-item responses.
   - Mood detection latency: <500 ms on GPU, <2 s on CPU.

2. Reliability
   - System availability ≥ 99% with graceful degradation for model unavailability.

3. Security
   - All protected endpoints require Firebase ID token validation.
   - Firestore security rules enforce uid-based access controls.

4. Scalability
   - Support ≥ 1,000 concurrent requests with horizontal scaling and stateless Flask instances.

5. Maintainability
   - Modular code organization (routes/services/ml/persistence) with unit- and integration-test coverage.

6. Privacy
   - Per-user data isolation; option to disable LLM insights for privacy-conscious users in `config.yml`.

7. Reproducibility
   - Model versions, config settings and experiment metadata recorded for each run.

Supplemental non-functional subsections (performance, safety, security, software quality attributes) are presented in subsequent files.
